Unix

Unix is the dominant family of operating systems for servers and web development. For this reason, it’s important to have a solid understanding of how Unix systems work and how to work effectively with them for basically anybody working in the web development space.

Many aspects of Unix are standardized (through the POSIX standard). This means that Unixes are very similar from a usage standpoint, and that programs are often available for more than one Unix.

The Unixes you’re by far most likely to encounter are macOS (formerly OS X) and Linux. Linux is the dominant open-source Unix kernel and OS family, especially in the server space (but also in e.g. mobile phones — Android is a Linux OS). macOS, which is the most popular Unix OS for personal computers, is a proprietary OS that only runs on hardware produced by Apple. The main difference between Linux and macOS from a usage standpoint (as long as you’re not working with kernel APIs) is that they tend to come with slightly different variants of many tools (Linux uses the GNU suite of utilities, whereas macOS uses the BSD suite).

In this section, we present some core Unix concepts and utilities, and discuss how to learn to work effectively with Unix (and in particular, the Unix shell).

On Unix systems, users optionally belong to a set of groups, and each file and directory also has an associated group. On Unix systems, permissions are granted on a file-by-file basis, and separately for the owner, the group, and others (everybody else). There are three distinct permissions that can be granted for files: read (the file may be read), write (the file may be written to), and execute (if it’s a program, the file may be executed). For directories, the same three permissions are used, but they have different meanings (read means files in the directory may be listed, write means files in the directory may be created/deleted/renamed, and execute means the directory may be searched).

SSH

SSH (Secure Shell) is the protocol and tool most commonly used to securely access (to run an interactive shell or command) remote hosts over a network. It supports multiple authentication methods, but by far the most common is public-key cryptography (see the ‘Cryptography’ section).
