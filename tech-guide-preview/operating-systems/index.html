Operating Systems

The software that handles communication with and management of a computer’s hardware is called a kernel. Kernel are bundled with various utilities, graphical interfaces, etc. as Operating Systems (OSs).

The dominant OSs are Windows, macOS (formerly OS X), and Linux. Linux isn’t really a single OS, but rather a family of OSs that all use the Linux kernel.

Linux and macOS both belong to the Unix family of OSs. This means that they’re very similar from a usage and application development perspective, and any given piece of software is typically available for both.

Kernels provide application programming interfaces (APIs) for programmers/programs to interact with. Calls made to these APIs are called system calls (or syscalls).

Kernel APIs are built around a set of abstractions, which help provide a more convenient interface than dealing directly with the underlying hardware. The following abstractions are especially important:

  • Processes: A process is an instance of a running program. Processes can make syscalls to launch (or spawn) other processes â€” these are called child processes of the original process. Each process has a unique Process ID (PID).
  • Threads: A thread is a sequence of instructions running independently as part of a program. Processes start with one thread, and can spawn more threads via syscalls. Used to make programs to multiple things at the same time.
  • Files: A file is a chunk of data, given a filename (sometimes containing a filename extension like .html) and stored on disk. Files can be created, moved, deleted, and opened (for reading and/or writing) through syscalls.
  • The file system: a big data structure into which all files on a computer are organized. In Unix, the file system is organized as a single rooted tree of directories (or folders); the top-level directory of which is called the root directory and denoted /. In Windows, the file system consists of multiple rooted trees (with root directories C:, D:, etc.). Note: Confusingly, the word ‘file system’ is also used to refer to the format used to serialize files on disk.
  • Sockets: Sockets are channels that can be opened either between processes running on the same machine (e.g. Unix sockets) or between hosts on different machines (network sockets) for the purpose of sending data back and forth.

The following abstractions are also good to be aware of:

  • Virtual memory: A chunk of memory allotted to a process to store data in. If all available memory is exhausted, the OS sometimes transparently relocates parts of the virtual memory to disk (this is called paging, and the place where the data is stored is called a swap file or swap partition).
  • Environment variables: A set of values, addressed by keys, used to pass configuration information to processes. Each process has an associated set of environment variables, which are usually ‘passed down’ to its child processes.
  • Signals: notifications sent from the OS to a process, typically to tell it to stop running.

In order to be executed by the OS, a program must first be compiled (translated to machine code using the appropriate instruction set) and stored as an executable binary file (in a format understood by the OS). Programs can delegate some work to shared libraries, which are chunks of code written by somebody else, that have been compiled and are stored somewhere on the file system.

Modern OSs are multi-user; they have multiple user accounts, often but not always associated with specific end users (humans). Each file, directory and process is owned by some user account (typically the account that created it), and users can not access files or directories owned by other users unless they have been granted permission to do so. There’s also a special user called the superuser (also called root user or administrator), which always has permission to do whatever it wants. OSs typically have mechanism that allow end users to execute specific commands as the superuser.

OSs usually ship with both a graphical user interface (GUI) and a command-line interface (CLI) called a [shell](https://en.wikipedia.org/wiki/Shell_(computing). Most people who use computers never interact with a shell, but knowing how to use a shell well is an essential skill for power users.
